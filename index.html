The Problem · Documented by GAO

FSA shipped a broken system — then stopped tracking whether it was fixed.

🔴 FAFSA Processing System (FPS)
  • 9 of 25 contractual requirements never deployed
  • 30% of forms had errors Mar–Apr 2024
  • SAI formula omitted asset data — 5M incorrect aid offers
  • By May 2025, FSA stopped tracking the missing requirements entirely
🔴 Quality & Oversight Failures
  • Test cases had no traceability to requirements — impossible to verify what worked
  • No user testing plan for future FPS updates
  • Contractor performance never independently validated
  • Key staff lacked required certifications for the dev methodology used
"FSA was unable to provide the status of the complete system... they were no longer tracking requirements." — GAO-25-107396, September 2025

Agent Architecture

Each federal integration becomes an MCP server — defined inputs, validated outputs, full audit trail.

🤖 Orchestrator Agent

FAFSA event → parallel dispatch → collect results → assemble ISIR → route to FAA with audit package.

PM owns: state machine design, timeout handling, retry logic spec

📋 Audit Agent

Every call — input, output, timestamp, confidence — logged immutably. FERPA compliance and GAO defensibility.

🔔 Notification Agent

Real-time status to students and FAAs. Eliminates the 3.5M unacknowledged forms problem.

🏦 IRS MCP Server

In: SSN + consent token · Out: FTI validated · SLA <2s · Fallback: manual queue

🔍 SSA MCP Server

In: Name + DOB + SSN · Out: confidence 0–1 · SLA <1s · ≥0.85 auto-approve

🏛️ DHS MCP Server

In: Citizenship + residency · Out: eligibility flags · SLA <3s · Edge cases → manual

📊 Eligibility Calculation

In: Validated IRS + SSA · Out: SAI + Pell · Formula: versioned, tested, auditable

The Problem · Documented by GAO
FSA shipped a broken system — then stopped tracking whether it was fixed.
🔴 FAFSA Processing System
  • 9 of 25 requirements never deployed
  • 30% of forms had errors Mar–Apr 2024
  • SAI formula omitted asset data — 5M wrong aid offers
  • By May 2025 FSA stopped tracking the missing requirements
🔴 Quality & Oversight
  • Test cases had no traceability to requirements
  • No user testing plan for future updates
  • Contractor performance never independently validated
  • Key staff lacked required certifications
"FSA was unable to provide the status of the complete system... they were no longer tracking requirements." — GAO-25-107396, September 2025

Agent Architecture
Each federal integration becomes an MCP server with a defined contract.
🤖 Orchestrator Agent

FAFSA event → parallel dispatch → collect results → assemble ISIR → route to FAA with audit package.

PM owns: state machine, timeouts, retry logic

📋 Audit Agent

Input, output, timestamp, confidence logged immutably. FERPA + GAO defensibility. Non-negotiable.

🔔 Notification Agent

Real-time status to students + FAAs. Eliminates the 3.5M unacknowledged forms problem.

🏦 IRS MCP Server

In: SSN + consent_token · Out: FTI validated schema · SLA <2s · Fallback: manual queue

🔍 SSA MCP Server

In: Name + DOB + SSN · Out: confidence 0–1 · SLA <1s · ≥0.85 auto-approve

🏛️ DHS MCP Server

In: Citizenship + residency · Out: eligibility flags · SLA <3s · Edge cases → manual

📊 Eligibility Calc

In: Validated IRS + SSA · Out: SAI + Pell · Formula: versioned, tested, auditable

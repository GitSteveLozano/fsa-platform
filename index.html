The Current State

What the GAO Found

Two GAO reports (2023, 2025) documented specific, preventable failures:

🔴 FAFSA Processing System (FPS)

  • Deployed Dec 2023 with 9 of 25 requirements undeployed
  • 30% of forms had errors (Mar–Apr 2024)
  • SAI formula missing asset data for dependent students
  • By May 2025: FSA stopped tracking those requirements

🔴 Quality & Oversight Failures

  • Test cases had no traceability to requirements
  • No user testing plan for future updates
  • Contractor performance data not validated
  • Staff lacked required certifications for dev methodology
"FSA was unable to provide the status of the complete system... they were no longer tracking requirements." — GAO, Sept 2025
These aren't hypothetical risks — they're documented failures with real students affected.

Agent Architecture

The MCP-Connected Agent Layer

Each federal integration becomes a structured MCP server. The Technical PM owns the API contracts — not the code.

Orchestrator Agent

FAFSA event → parallel agent dispatch → collect results → assemble ISIR → route to FAA with audit package.

PM owns: State machine, timeout handling, retry logic spec

Audit Agent

Every call — input, output, timestamp, confidence — logged immutably for FERPA compliance and GAO review.

Notification Agent

Real-time status to students + FAAs. Eliminates the 3.5M-unacknowledged-forms problem.

🏦 IRS MCP Server

In: SSN + consent token · Out: FTI (validated schema) · SLA: <2s · Fallback: manual review queue

🔍 SSA MCP Server

In: Name + DOB + SSN · Out: Identity confidence (0–1) · SLA: <1s · Threshold: ≥0.85 auto-approve

🏛️ DHS MCP Server

In: Citizenship/residency · Out: Aid eligibility flags · SLA: <3s · Edge cases: manual review

📊 Eligibility Calculation

In: Validated IRS + SSA data · Out: SAI + Pell eligibility · Formula: versioned, tested, auditable
